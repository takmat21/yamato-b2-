# ヤマトB2 伝票変換ツール

Amazon・Yahoo!ショッピング・自社Webショップ（BASE系）の注文ファイルを、
**ヤマト B2クラウドの「外部データ取り込み基本レイアウト」（95列）CSV** に変換する
ブラウザ完結ツールです。インストール・マクロ・サーバー不要。HTMLファイルを開くだけで動きます。

> もともとは Excel マクロで行っていた作業を、セキュリティ仕様変更でマクロが使えなくなったため、
> ブラウザツールに置き換えたものです。

## 🔒 プライバシー

**注文データは外部に送信されません。** ファイルの読み込み・変換・CSV出力はすべて
お使いのブラウザ内（JavaScript）だけで完結します。サーバーへのアップロードは一切ありません。

ご依頼主（出荷元）の固定情報は画面上の入力欄で設定でき、その内容はブラウザ内（localStorage）に
のみ保存されます。

## 使い方

1. ページを開く。
2. （初回のみ）「**ご依頼主・固定値の設定**」を開き、自社の値を入力（初期値はサンプル）。
3. Amazon / Yahoo! / 自社ショップの注文ファイルを **ドラッグ＆ドロップ**（複数まとめてOK・種類は自動判別）。
4. 必要に応じてオプションを調整し「**変換する**」をクリック。
5. プレビューを確認（セルは直接クリックして修正可能。赤いセルは要確認）。
6. 「**CSVをダウンロード**」して B2クラウドに取り込む、または「**コピー**」してExcelテンプレートに貼り付け。

### 対応チャネル（自動判別）

| チャネル | 形式 | 判定キー（1行目ヘッダー） |
|---|---|---|
| Amazon | TSV（.txt） | `order-id` を含みタブ区切り |
| Yahoo!ショッピング | CSV | `OrderId` と `ShipRequestDate` |
| 自社 / BASE | CSV | `注文ID` と `配送先` |

文字コードは自動判定（Amazon=UTF-8、Yahoo/自社=Shift-JIS が多い）して読み込みます。

### オプション

- **ヘッダー行を付ける**（既定オフ）
- **品名を略号化**（既定オン）: アダルト→A、ピンク→P
- **同一宛先を1伝票にまとめる**（既定オン）
- **住所・建物名のスペースを詰める**（既定オン）

## 変換ルール（要点）

- **出荷予定日（E列）** = お届け予定日 − 2日。ただし本日より前になる場合は本日に繰り上げ。
  お届け予定日が空欄なら本日。（締切による翌日繰り上げは行わない）
- **品名（AB/AD列）**: 定型語・スペック表記を除去し略号化。1種類なら品名1＝略号名・品名2＝数量。
  2種類以上なら品名1＝「略号＋数量」を区切りなしで連結（例 `AS1AL1AM1`）・品名2＝合計数量。
- **住所（L/M列）**: スペースを詰める。ただし前後が両方数字のときのみハイフン `-` に置換。
  BASEの住所2が番地（数字・ハイフンのみ）なら住所へ結合、建物名なら建物欄へ。
- **電話番号（I列）**: 国際表記 `+81` / `＋81` / 先頭 `81` を `0` に変換（ヤマトは+81でエラー）。
- **同一宛先の統合**: 郵便番号・住所・建物名・氏名・電話・お届け日が一致する明細を1伝票に統合。
- **配達時間帯（G列）**: 08:00→0812 / 14:00→1416 / 16:00→1618 / 18:00→1820 / 19:00→1921。
- **代引（B/AH列）**: Amazon=COD、Yahoo/BASE=支払い方法に「代引」を含む場合に金額をセット。
- **お届け予定／完了メール（AV〜BB列）**: Yahoo!・自社（BASE）は送信（利用区分=1）。
  **Amazon は送信しない（利用区分=0）** — Amazon／ヤマト側が配送通知を行い、出荷元からの
  直送メールを受信拒否にしているお客様が多いため。

固定値（全行共通）はクール区分=1（冷凍）、荷扱い1=ナマモノ、お届け予定／完了メール利用、
およびご依頼主名・電話・郵便番号・住所・請求先顧客コード・運賃管理番号（画面で設定可）。

## 出力

- **CSVダウンロード**: Shift-JIS・95列。B2クラウドの「送り状発行 → 外部データ取り込み」に取り込み。
- **クリップボードにコピー**: Excelテンプレ貼付用。HTMLの表としてコピーするため、貼り付け直後から
  全文表示・先頭0欠落なし（失敗時はタブ区切りにフォールバック）。

## 技術構成

- 単一HTML（`index.html`）＋ Shift-JIS変換ライブラリ
  [encoding-japanese](https://github.com/polygonplanet/encoding.js)（`vendor/encoding.min.js` に同梱）。
- サーバー・ビルド・npm 不要。オフラインでも動作（CDN非依存）。

```
.
├── index.html              … 本体
├── vendor/
│   └── encoding.min.js     … encoding-japanese v2.2.0（同梱）
└── README.md
```

## 公開方法（GitHub Pages の有効化）

1. このリポジトリの **Settings → Pages** を開く。
2. **Build and deployment → Source** を「Deploy from a branch」に。
3. **Branch** を `main` / `/ (root)` に設定して **Save**。
4. 数十秒後、`https://takmat21.github.io/yamato-b2-/` で公開されます。

> このツールへ反映するには `main` ブランチにマージしてください。
> ローカルのファイルを直接ブラウザで開いても同じように動作します。

## 動作確認（受け入れテスト）

3チャネルのサンプルファイルを読み込み、以下を確認します。

- 同一宛先の明細が1伝票に統合される（同一注文の2商品が1伝票・品名2に合計数量）。
- お届け 6/17 → 出荷 6/15（本日6/15の場合の2日前ルール）。お届け 6/16 → 6/14 は過去日なので本日に繰り上げ。
- `+81…` の電話が `0…` に変換される。
- 住所のスペースが詰まり、数字どうしはハイフンになる。
- CSVが Shift-JIS で出力され B2 に取り込める。コピーをExcelに貼ると各セルに正しく展開・全文表示される。
